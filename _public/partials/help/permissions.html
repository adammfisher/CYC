Permissions

Cyclotron has integration with LDAP or Active Directory for user authentication. Users can login with their domain username/password. Sessions are kept active by using Cyclotron, but will periodically expire for security reasons.

Dashboards are unsecured by default, but can optionally be secured to a list of Editors and/or Viewers. These are both optional, and if omitted, allow everyone to edit and view, respectively. Viewing of Dashboards cannot be restricted unless editing has also been restricted.

Permission model

Both Editor and Viewer permissions are based on LDAP/Active Directory Users and/or Groups. In both cases, one or more Users and/or Groups can be added to the Dashboard, restricting permissions to the specific Users listed, or Users that are members of the specific Groups listed. Nested Groups are not supported!

There are several types of Groups: Security Groups, Distribution Lists, Groups. Any of them can be used interchangeably in Cyclotron.

Edit Permissions

Creating and Editing Dashboards requires the current user to be logged in. This is for audit purposes and tracking who made which Dashboard changes. By default, Dashboards can be edited by any logged-in user. This refers specifically to saving changes to a Dashboard, not merely viewing the Dashboard definition in the Editor (which falls under View Permissions)

Editing can be restricted by adding one or more Editors to the Edit Permissions box on the Details page of the Dashboard Editor. Auto-complete is enabled after typing three characters.

View Permissions

View Permissions covers the ability to see a Dashboard, including viewing the Dashboard definition, exporting the Dashboard, etc. By default, all users have view permission for all Dashboards, even if they cannot edit the Dashboards.

Viewing can be restricted by adding one or more Viewers to the View Permissions box on the Details page of the Dashboard Editor. Auto-complete is enabled after typing three characters.

Anyone with Edit Permissions automatically inherits View Permissions. Note that this means View Permissions are ignored unless Edit Permissions are set, as everyone can edit by default, and thus everyone can view as well.

Summary of Actions

Action Requires Login Default Permission Possible Permission
Searching for Dashboards Everyone N/A
Creating a Dashboard Logged In N/A
Editing a Dashboard (Saving Changes) Everyone Users and/or Groups
Viewing Dashboard Editor Only if View restricted Everyone Users and/or Groups
Viewing a Dashboard Only if View restricted Everyone Users and/or Groups
Exporting a Dashboard Only if View restricted Everyone Users and/or Groups

REST API

The REST API implements the same permission model as the UI, requiring login and session keys on all endpoints requiring authentication.
