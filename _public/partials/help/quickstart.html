Quick Start

Before you start building a Dashboard with Cyclotron, make sure you know two things:

  1. What data do you want to display, and is it available?
  2. How do you want to visualize it?

Knowing the answer to these questions will make it easier to get started with Cyclotron and avoid unnecessary effort. It's also recommended to review the Example Dashboards, to get a idea of what Cyclotron's capabilities are.

Creating a Dashboard

Before creating a Dashboard, you must login. Click on the Login button on the home page, or on the top-right of the header. After logging in, the header will show the login status, and display your Gravatar image (if configured).

Click on the New Dashboard button on the home page, or on the top-right of the header. The Dashboard Editor will open with an empty configuration.

Name
Dashboards cannot be renamed after they have been created. Choose a name wisely.

Enter a name for the dashboard in the Name textbox. This name will be used in the URL to access the Dashboard. The name must be unique and alphanumeric—special characters are not allowed, and spaces will be converted automatically into dashes.

Tags

Tags are keywords used to organize and search for Dashboards. For example, example Cyclotron dashboards are all tagged with cyclotron-examples so they can be easily discovered. In the Tags box, select one or more existing Tags, or enter a new Tag. This step is optional, and more tags can be added or removed later.

Edit Permissions and View Permissions

By default, Cyclotron dashboards are open to everyone for viewing and editing. These two properties provide the means to restrict editing or viewing to specific user(s) and group(s). For more information, see the help section Permissions.

Description

The description property is an optional note displayed in search results and at the top of the Dashboard editor.

Theme

Several predefined themes are available, which change the color and styling of the Dashboard and Widgets. If left unset, Cyclotron will use the default theme ({{ config.dashboard.properties.theme.default }}). Pages and Widgets can override the Dashboard theme, but will inherit this property by default.

Additional Properties

Cyclotron has many optional properties, so the Additional Properties dropdown contains those less-frequently needed. Selecting any of the properties in this dropdown adds to the Dashboard so it can be configured.

Saving the Dashboard

At this point, click the green Save button at the top of the page. This will create the first revision of the new Dashboard. At this point, the Dashboard is available for anyone to view. Click the blue Preview button to open the Dashboard in a new window.

Revisions

Each time the Dashboard is modified and saved, the Revision number increments. This is displayed at the top of the Dashboard Editor. A copy of the Dashboard is saved as well, so older Revisions can be viewed or restored. To view older Revisions, click on the Revision number link at the top of the Dashboard Editor. This enables scrolling backward/forward through the Revision history. The Preview button will preview the currently selected Revision as well.

To restore an older Revision, scroll backwards in the history to select the correct Revision, then click Save. This will save a copy of the older Revision as a newer Revision, and increment the Revision number.

Viewing the Dashboard

The Dashboard can be viewed in several ways:

  • Searching for the Dashboard by name or Tags, and selecting the Dashboard
  • Clicking the Preview link in the Dashboard Editor
  • Directly navigating to the Dashboard URL

Dashboard navigation controls are available by hovering the mouse over the bottom center of the screen. If a Dashboard has more than one page, pages can be scrolled through using the left and right arrow buttons.

The Dashboard can be saved and previewed at any time during development, and as frequently as needed.

Adding a Data Source

Before the Dashboard will display anything, it needs to load some data via a Data Source. Open the Dashboard Editor and click on Data Sources in the sidebar.

This page lists all the Data Sources in the Dashboard. Click Add Data Source to create a new, empty Data Source. The Data Source type and name will be shown in the list on this page. The sidebar also lists each Data Source for rapid navigation.

Data Sources can also be cloned or removed, and moved up and down. The order of Data Sources does not have any effect on the Dashboard and is cosmetic only.

To edit the new Data Source, click on its name, or select it in the sidebar menu.

Type

The type property has a dropdown with all supported Data Source types. Changing the selected type will change what properties are available in the editor. Select the json Data Source type, which is used to load JSON documents.

The url property is required, and specifies the web location to download; it can be a static file or a web service. Here's a sample JSON file: http://pastebin.com/raw.php?i=4yCbDza4

This sample JSON file can be used without any other configuration, as it is already in the correct format and does not require authentication. Enter this URL in the url property.

Name

The name is used by Widgets to reference this Data Source, so it should be a unique, meaningful name.

Data Source Configuration

The remaining configuration will be largely specific to which Data Source type is selected. For more information on Data Sources, refer to the help section on Data Sources

Adding a Page

After the Data Source is added, a new Page must be added to display the data. In the Dashboard Editor, click on Pages in the sidebar.

This page lists all the Pages in the Dashboard. Click Add Page to create a new, empty Page. The Page name and list of widgets will be shown in the list on this page. The sidebar also lists each Page for rapid navigation.

Pages can also be cloned or removed, and moved up and down. The order of Pages determines the order they are displayed when viewing the Dashboard.

To edit the new Page, click on its name, or select it in the sidebar menu.

Name

The name is displayed in URL when viewing this page. It is optional—a default name will be generated from the page number if omitted.

Layout

The Page Layout is important for determining how Widgets appear on the Dashboard. Cyclotron uses a grid-based layout, so each page has a configurable number of rows and columns. Widgets can span one or more rows/columns, and thesize of each row and column will be adjusted to fit in the browser.

The Page Layour defaults to a 2x2 grid, which can contain 4 1x1-sized Widgets. Change Grid Columns to 1, and Grid rows to 1 as well. This will fit a single Widget.

Widgets

By default, the Page has no Widgets on it. Click the blue Add Widget to create a new Widget on this page. The Widget type (and title) will be shown in the list on this page. The sidebar also lists each Widget in the current Page for rapid navigation.

To edit the new Widget, click on its name, or select it in the sidebar menu.

Adding a Widget

Type

Each Widget on the Dashboard must be one of several supported Widget Types. By default, a Widget has no Type. In the Widget Type dropdown, select Table. Changing the selected Widget Type will change what properties are available in the editor.

Table is a good Widget to begin with, because its default behavior is to show all the rows and columns of the Data Source, without needing any configuration.

Data Source

To reference the Data Source created earlier, select the Data Source name from the Data Source property dropdown. This connects the Widget to the results from the Data Source.

Layout

Each Widget has a layout configuration, which is used in conjection with the Page Layout to determine the size of the Widget. The default Widget size is 1x1. Since the Page Layout is set to 1x1, set this Widget to 1x1 to match (or leave it unset to use the default).

Save and Preview Dashboard

At this point, the Dashboard should have a Data Source and a connected Table Widget. Save and Preview the Dashboard now, to see the result.

Continuing Development

This was a basic introduction to creating a very simple Dashboard. From here, more Data Sources or Widgets can be added and configured.

For more information on Widgets in general or specific Widgets, refer to the help section on Widgets.
